her.ai — Talk to Anyone from Your WhatsApp History

python mcp rust ai rag whatsapp cursor

The Idea

She blocked you. She's gone. But her chats still haunt you — that one WhatsApp export sitting in your Downloads folder, thousands of messages you'll never get a reply to again.

So you do what any reasonable engineer would do — you build an AI version of her.

That's her.ai. Feed it a WhatsApp chat export, and it texts back exactly the way that person does. Same Hinglish, same slang, same emoji game. Not a generic chatbot — an AI that actually sounds like them because it learned from thousands of real messages.

Two Ways to Run It

her.ai has two approaches. Pick whatever fits your setup:

MCP Server (Cursor) — recommended

Plugs into Cursor IDE via MCP. No separate API keys, no rate limits to worry about. Just type me: and start chatting. If you're already using Cursor, this is the way to go.

Gemini CLI

Standalone Python script that hits the Gemini API directly. Works outside of any editor but needs a Google API key and hits free-tier rate limits after a few minutes.

The MCP Server

The MCP server plugs into Cursor and uses whatever model the editor already provides. No API keys to manage, no rate limits to hit. You just type me: followed by your message, a Cursor Rule auto-triggers the MCP tool, and the model replies in character.

How MCP works here

The server doesn't call any LLM itself. It parses the chat, builds a system prompt with style examples + history, and returns it as context. Cursor's model does the actual generation. You still pay for Cursor, but you skip the hassle of managing separate API keys and rate limits.

The server maintains session memory across turns — a simple in-memory list of (speaker, message) tuples. Each reply you generate gets passed back via previous_persona_reply on the next call, so the conversation stays coherent.

The Gemini CLI

If you don't use Cursor, there's a standalone CLI that hits the Gemini API directly. It converts each message into a HumanMessage or AIMessage, loads the last 600 as context, and chats in a loop. The persona is auto-detected — whoever isn't you is the one it mimics.

# Convert chat history into LangChain messages for sender, text in parsed_msgs: if sender == responder_name: lc_messages.append(AIMessage(content=text)) elif sender == sender_name: lc_messages.append(HumanMessage(content=text))

Works well, but you'll hit Gemini's free-tier rate limit (20 requests/minute) after a few minutes of chatting. That's actually what led to building the MCP server in the first place.

The Chat Parser

WhatsApp exports are deceptively annoying to parse. The format varies by locale, OS, and app version. Indian exports use DD/MM/YY, HH:MM am (lowercase am/pm) while others use uppercase AM/PM or 24-hour time.

LangChain's built-in WhatsAppChatLoader choked on the format entirely. Wrote a custom regex parser:

pattern = re.compile( r'^[\d/]+,\s+[\d:]+\s+[ap]m\s+-\s+([^:]+):\s*(.+)$', re.IGNORECASE )

It also filters out <media omitted>, deleted messages, and encryption notices. The Rust side of the codebase handles converting between different WhatsApp export formats — useful when the export uses square-bracket timestamps instead of the dash-separated format.

What I Learned

  • Style examples > descriptions — Telling the LLM "they use Hinglish" doesn't work. Showing it 60 real messages from that person does.
  • Parsers are never simple — Lowercase am/pm broke the parser silently. Zero messages parsed, no errors. Case-insensitive regex saved it.
  • MCP as a middle ground — You still pay for Cursor's model usage, but you skip separate API key management and rate limit headaches. It's not free — it's just more convenient than raw HTTP calls to Gemini/OpenAI.
  • Style > Knowledge — 60 real message samples teach the LLM more about someone's texting vibe than any written description could.
